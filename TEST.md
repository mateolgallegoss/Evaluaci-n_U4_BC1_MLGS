## 1. Conceptos Generales
### ¿Qué es un archivo Markdown y para qué sirve? (Responde en un párrafo corto)
Un archivo Markdown es un archivo de texto que usa una sintaxis sencilla para dar formato al contenido, siendo una simplificación de HTML. Usada prácticamente como una ley para documentar el contenido de un repositorio en API's como Github o GitLab.

### ¿Cuál es el propósito del archivo README.md en un repositorio? (Responde usando texto en cursiva).
*El archivo README.md se presta como una introducción de tu repositorio, sirviendo para explicar qué hace, cómo se usa, cómo instalarlo y otra información relevante de interés para facilitar el acceso al repositorio, ayudando a entender su propósito.*

## 3. Flujo de Trabajo en GitHub Desktop
**¿Qué diferencia hay entre hacer un "Commit" y hacer un "Push"?**
* **Commit:** (Registra y guarda los cambio del directorio en el área de trabajo o el stage para subirlos al repositorio. Siempre va documentado con una descripción sobre los cambios.)
* **Push:** (Se encarga de subir los commits al repositorio para que otras personas conozcan de forma rápida cuáles son los cambios y puedan editar los archivos.)
 
**¿Para qué sirve el botón "Fetch Origin"?**
> Permite ver los cambios realizados por otras personas sin subir las actualizaciones al repositorio.
